在 Hexo 中使用 Shiki 进行代码高亮
✨小透明・宸✨
2025-04-30 16:04:56

如果想要成品的话可以直接翻到最后～

我没少折腾过这个 Hexo 博客的代码高亮，一开始用的是 highlight.js，后来在自己写现在在用的这个主题的时候用的是 PrismJS，这两个都是 Hexo 自带支持的代码高亮库。

最近在进一步修改博客的深色模式，想要让代码高亮在浅色和深色模式下使用不同的主题。一开始的想法是选两个 PrismJS 主题各自放进 @media (prefers-color-scheme: light/dark) 然后“混搭”出一个 CSS，在选择主题的时候发现 PrismJS 已经很长一段时间没有更新了（实际上是在准备下一个大版本 v2），在相关的讨论里有人推荐了 Shiki。看了一下演示和文档，它的代码高亮更精确，支持的语言、主题、功能也比 PrismJS 丰富得多，因此就决定替换了。

注意到在 Shiki 高亮后的代码中，颜色等样式都是直接写在每个元素的 style 属性中，如果代码较多的话我们会希望从这些重复的内联样式中节省一些生成的 HTML 片段的大小（已有的那些插件都没有考虑这个）。Shiki 的 transformerStyleToClass 可以将内联样式转换为唯一的 class，这样就可以收集整个页面所有代码块高亮后的 class 然后进行去重处理。

getClassRegistry 返回的是类型为 Map<String, Record<String, String> | String> 的内联样式，不过这可以很容易统一成 Map<String, String>。

不进行去重的话可以加一个 <style> 直接写入这些样式。但是如果要收集一个页面中所有代码块的样式的话……Hexo 的代码高亮并不会传入当前高亮的代码来自于哪个文件。我想到的做法是先将样式以 JSON 格式再包一层 Base64 写入代码块后面的注释中，然后加一个 hexo.extend.filter.register('after_render:html')，在这里使用正则表达式提取之前写在注释中的样式，然后进行去重，同时去除这些临时添加的注释：
