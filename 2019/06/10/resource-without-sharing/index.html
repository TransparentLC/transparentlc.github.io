资源下载

“为什么不使用百度网盘？”

在下载资源前，小透明还是要做一些说明( ｡･_･｡)

现在一提到“发资源”，大家第一时间想到的应该都是百度网盘吧？

百度网盘凭借人均 2TB 的容量、方便的“分享”功能、当然最重要的是有钱可烧，在数年前的“网盘大战”中成为了最后的优胜者，今天的中文互联网上到处都是通过百度网盘分享的资源。

（“废话”里面提及的公众号也是百度网盘的忠实用户，甚至还将速盘之类的第三方客户端用来充当自己为“大会员”们“特别准备”的“网盘高速下载器”←_←）

但百度网盘在通过（早期的）高质量的服务吸引了绝大多数用户后，转而通过 10KB/s、“账号黑名单”这类严重降低用户体验的手段逼迫用户开通会员，作为商业行为可以理解，却给用户造成了极大不便。

所以小透明不会把这里的资源上传到百度网盘，而是上传到一个名为 Catbox 的国外网站～（ ‘-ωก̀ ）

Catbox 提供任意类型的文件上传服务，上传文件即可获得形如 https://files.catbox.moe/******.zip 这样的直链，（在站长决定关闭服务器之前）文件永久存储，而且下载不限速（由于物理原因可能会慢一点？），不过网站对于上传的文件有一些处理：

  • 单个文件不能超过 200MB。
  • 上传文件后，直链中的文件名会更改成随机的六个字母和数字，但保留源文件的扩展名。

由于以上处理的存在，小透明会将每个资源打包成每个文件 195MB 的分卷压缩包，然后将每个分卷单独再打包成压缩包，然后上传～

压缩包的下载链接会全部存放在如下格式的 JSON 文件中：

{
    "package": [
        {
            "url": "https://files.catbox.moe/******.zip",
            "hash": "0123456789ABCDEF0123456789ABCDEF"
        },
        {
            "url": "https://files.catbox.moe/******.zip",
            "hash": "0123456789ABCDEF0123456789ABCDEF"
        },
        ...
    ],
    "split":  true,
    "filename":  "..."
}
名称说明
package由包含 url 和 hash 的项组成的数组。
url每个压缩包的下载地址。
hash每个压缩包的文件 MD5，可以用来校验。
split资源是否为分卷压缩包。如果是的话则需要先从压缩包中解压出分卷，再从分卷压缩包中解压出资源，否则可以直接从压缩包总解压出资源。
filename分卷压缩包的文件名。

下载方法

自动下载（Windows 限定）：

  1. 从这里下载一个 PowerShell 脚本，右键“使用 PowerShell 运行”。（脚本里没有私货，不信的话你自己打开看看((꜆꜄•௰•)꜆꜄꜆ ）
  2. 在“下载地址”处右键复制链接，在 PowerShell 的窗口中右键粘贴，回车。
  3. 等待压缩包下载完成，自动解压，获得资源。

手动下载：

  1. 点击“下载地址”保存一个 JSON 文件，用记事本打开（也有可能是浏览器自己已经直接打开了），找到所有 https://files.catbox.moe/******.zip 的链接，用浏览器、IDM、迅雷之类的工具下载。
  2. 解压所有下载的压缩包。完成这一步可能已经获得资源了，否则请继续完成下一步。
  3. 找到扩展名为 *.zip.001 的分卷压缩包，用压缩软件解压，获得资源。
